### **The Claude Prompt: "SMM Platform Architect & Automation"**

**System Role:**
"You are acting as a Senior Full-Stack Engineer and Business Systems Architect. I am launching an SMM (Social Media Marketing) panel reselling business in Nairobi, Kenya. I need a robust, automated infrastructure that bridges a frontend interface with a specific SMM Provider API."

**The Context:**

* **The Backend:** I am using an SMM Master Provider API (SMMWiz).
* **The Frontend:** I want a high-performance, minimalist/brutalist Web UI and a Telegram Bot for order placement.
* **The Payment:** I am using MegaPay (linked to my local bank) for M-Pesa transactions.
* **Team:** My partner handles sales/DMs; I handle the technical automation.

**The Technical Requirements:**

1. **API Integration:** Create a Python/TypeScript wrapper for the SMM Provider API (actions: `add`, `status`, `services`, `balance`).
2. **Order Logic:** Implement a 'Safe-Growth' drip-feed logic for large orders (e.g., splitting 5k followers over 10 days using 24-hour intervals).
3. **Telegram Bot:** Build a bot using `python-telegram-bot` that:
* Lists service packages (IG, TikTok, YouTube).
* Accepts a profile link and validates it.
* Integrates a payment confirmation step (MegaPay webhook or manual approval toggle).


4. **Database:** Design a simple PostgreSQL schema to track `Orders`, `Clients`, `Transactions`, and `RefillStatus`.

**Specific Service Packages to Program:**

* **TikTok Viral Starter (KES 1,500):** 2k Followers + 5k Views + 200 Likes.
* **IG Business Boost (KES 1,500):** 1.5k HQ Followers + 300 Likes.
* **YouTube Kickstart (KES 1,500):** 300 Subs + 1,000 Watch Views.
* **Follower Booster (KES 600):** 1k Followers + 30-day Refill.

**The Goal:**
"Generate the boilerplate for the Telegram bot and the API middleware. Focus on high uptime, clean error handling for API failures, and a sleek user experience that allows my partner to approve orders with a single click once payment is verified."
